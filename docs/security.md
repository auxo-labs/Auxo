# Auxo Security Features & Verification Matrix

This document tracks all security features, policies, and edge-case protections implemented or planned within the Auxo codebase. It lists the threat vectors addressed, implementation detail coordinates, and standardized Test IDs for automated test planning.

---

## Security Matrix

| Test ID | Security Feature | Status | Threat / Risk Addressed | Implementation Details | Verification / Test Strategy |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **SEC-01** | Row-Level Security (RLS) on Profiles | **Implemented** | An authenticated user inspecting network calls to query or modify credit balances of other developers. | `public.profiles` RLS policy restricting authenticated access to `SELECT` (read-only) for `auth.uid() = id`. | Attempt to retrieve profile records using a different user's UUID from the client side; assert query returns empty or throws access error. |
| **SEC-02** | Webhook Signature Verification | **Implemented** | Unauthorized users forging POST requests to the Stripe webhook route handler to credit their own accounts for free. | `stripe.webhooks.constructEvent` validation using `STRIPE_WEBHOOK_SECRET` in [route.ts](../src/app/api/webhooks/stripe/route.ts). | Send a mock POST webhook request without the `stripe-signature` header in production; assert response is `500` or `400 Webhook Error`. |
| **SEC-03** | Scratchpad Character Limits | **Implemented** | Users uploading massive text blocks (e.g. 50,000+ words) causing LLM request timeout crashes and expensive token bills. | `maxLength={15000}` (upgraded to `30000` for Developer Pack & BYOK users) in [editor.tsx](../src/components/editor.tsx). | Attempt to paste text exceeding the active tier limit (15k standard, 30k premium/BYOK); assert the text is truncated and warning indicator turns red. |
| **SEC-04** | LocalStorage Key Validation | **Implemented** | Hydration crashes or setting state injection attacks caused by importing corrupted browser storage values. | Safety validator checks in [useRoomSync.ts](../src/app/room/[id]/hooks/useRoomSync.ts) and [settings-modal.tsx](../src/components/settings-modal.tsx). | Set the `auxo-settings-provider` key to a corrupted string (e.g. `invalid_provider`) in browser dev tools and refresh; assert value defaults back to `premium`. |
| **SEC-05** | Service Role Token Separation | **Implemented** | Leakage of full read-write administrative credentials (`service_role_key`) to client-side browser files. | Decoupling standard client ([supabase.ts](../src/lib/supabase.ts)) from backend-only service clients (`supabaseAdmin`). | Verify the administrative client is never imported or initialized in files running `'use client'`. |
| **SEC-06** | Real-Time Sync Channel Authorization | *Planned* | Unauthorized third parties eavesdropping on client scratchpad broadcast text events by guessing Room UUID keys. | Implement custom JWT channel access signatures inside Supabase Realtime configs. | Attempt to connect a client socket to `room:[id]` channel without an active auth token; assert connection request is rejected. |
| **SEC-07** | Compile Endpoint Rate Limiting | **Implemented** | Automated scripts spamming the `/api/compile` endpoint, degrading server response latency. | Middleware rate limiting based on client IP addresses and user account IDs. | Execute 100 concurrent POST compilation requests from a single client node; assert subsequent requests receive `429 Too Many Requests`. |
| **SEC-08** | localStore Encryption for BYOK Keys | **Implemented** | Plain-text private developer API keys (Gemini/OpenAI/Claude) leaked during Cross-Site Scripting (XSS) client attacks. | Encrypting localStorage keys in memory using lightweight AES block-cipher/obfuscation keys. | Inspect local storage strings; assert keys are obfuscated/encrypted instead of readable text variables. |
| **SEC-09** | HTTPS BYOK Transmission Transit | **Implemented** | User private keys exposed during compilation request transit to backend proxies. | API routes are fetched exclusively over SSL/TLS (HTTPS) connection pipelines. | Verify that compile POST requests are strictly denied over non-TLS endpoints or automatically redirect to HTTPS. |
| **SEC-10** | Dynamic Value XSS Escaping | **Implemented** | Malicious script payloads injected into the raw markdown scratchpad executing in other users' browsers. | Built-in React rendering auto-escapes all strings rendered in JSX (e.g. preview tree gutters and code preview displays). | Paste `<script>alert(1)</script>` into the scratchpad; assert code is rendered as text and does not execute in DOM. |
| **SEC-11** | Public Client Access Scoping | **Implemented** | Public client credentials exploited to query administrative endpoints or write directly to Postgres. | Scope Supabase anon clients (`supabase.ts`) to restrict admin features or direct database queries. | Inspect client bundle keys; assert only anon key is loaded and service role token is restricted. |
